How It Works

This recommender takes the MovieLens dataset and turns each movie into a rich text profile by combining title, genres, movie tags, and optional overview metadata. The goal is to capture what each movie is about in a way that a machine can compare quickly.

Every movie record is cleaned, normalized, and tokenized so similar words and titles are compared consistently. The text for each movie is then transformed into a TF-IDF vector, which gives higher value to important keywords and reduces the impact of common words.

When you search for a movie, the app finds the movie in the dataset, then uses cosine similarity to compare its vector against all other movies. The best matches are the movies whose metadata is most closely aligned with the query movie in genre, tone, and user tag context.

Pipeline Overview

  • Load and preprocess MovieLens or Bollywood movie metadata
  • Normalize titles and clean genre labels
  • Aggregate user tags for each movie to enrich descriptions
  • Construct a combined movie metadata field for every film
  • Preprocess text with lowercasing, punctuation removal, and filtering
  • Build TF-IDF vectors using unigram and bigram features
  • Compute cosine similarity between the selected movie and all others
  • Return the top recommendations sorted by similarity score

Why this approach works

This is a content-based recommender, so it does not need user ratings to suggest movies. Instead, it relies on the shared movie description and metadata. That means it can recommend newer or less-rated movies as long as their text profile is meaningful.

The model is especially strong when movie similarity is driven by genre, keywords, and tag-based themes—so films like action thrillers, romantic comedies, or cult classics are easier to match accurately.
